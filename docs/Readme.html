1. The "Cold Start" Strategy (Retry Policy)

Hugging Face's Free Tier spins down models when inactive. The first request often fails with a 503 Service Unavailable.


The Solution: Instead of showing an error to the user, the GenerateImageUseCase implements a smart retry mechanism using retryWhen.

  • Intercepts 503 HTTP errors.
  • Applies exponential backoff (wait 1s, then 2s, then 4s).
  • Retries up to 3 times automatically before giving up.

2. Strict State Management

We avoid "null" checking hell by using a Sealed Interface for the result state.


sealed interface ImageGenerationResult {
    data object Loading : ImageGenerationResult
    data class Success(val image: Bitmap) : ImageGenerationResult
    data class Error(val message: String) : ImageGenerationResult
}

3. Dynamic API Routing

The networking layer is built to be provider-agnostic. By changing the Base URL and using dynamic path replacement, the app can switch between:

  • hf-inference (Free, Shared GPU, Slower)
  • fal-ai (Paid, Dedicated, Instant)

Currently configured for SDXL 1.0 via the internal router.
